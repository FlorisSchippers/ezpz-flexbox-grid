Using fractions

While the grid type is set to 'columns' by default, it's possible to change this to 'fractions'. This will enable you to think about the amount of space a cell should take within a grid section instead of constantly calculating the amount of columns you are working with.

Nearly all of a fraction based grid is identical to a column based grid, except for a few classes which are described below.

  • 1: Cells with specific sizes
  • 2: Horizontal cell offset

1: Cells with specific sizes

A .cell--2/3 class means that the element will take up two-thirds of the available space in the grid section. An element with the .cell--1/2 class will take up one-half of the grid section, etc.

2: Horizontal cell offset

When using a fraction based grid type, cells can be offset by adding a .cell--offset-{fraction} class to a .cell element. The second cell in the example has been offset by one-quarter.

Please note that cell offsetting is disabled by default and has to be enabled by setting the $ezpz-cell-offset variable to true.
